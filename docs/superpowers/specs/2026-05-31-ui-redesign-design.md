# 界面重设计：浅色主题 + 工具条常驻布局 — 设计文档

- 日期：2026-05-31
- 状态：设计方向已与用户确认（浅色 / 方案 C / 文件夹选择键等），待写实现计划
- 关联：在 `2026-05-30-invoice-total-amount-design.md`（发票价税合计功能）之上做**纯前端**视觉与布局重构；**后端零改动**
- Mockup（真实渲染存档）：`.superpowers/brainstorm/mockup-c-idle.html`、`mockup-c-run.html`、`mockup-a-run.html`（备选 A）

## 1. 背景与动机

当前界面为深色主题，用户反馈"有点乱"，集中在三方面（已确认）：

1. **空间分配失衡**：run 状态下顶部表单卡与结果区按 1:1 垂直堆叠，真正要看的「价税合计 + 表格」被挤在下半屏。
2. **元素细节杂乱**：「打开文件夹 / 返回说明」是裸文字链接、层级弱；汇总信息挤成一行散文字；完成后仍显示 100% 满进度条（冗余）；idle 状态右上角残留游离的「0 / 0」进度文字。
3. **整体观感不清爽**：希望换成浅色、更现代克制的整体风格。

**保留不动**：表格的数据组织（列、字段、失败行红色高亮、失败原因 hover）用户认为 OK。

## 2. 已确认的设计决策

| 决策点 | 结论 |
|--------|------|
| 整体风格 | **浅色亮色主题**（白底，接近财务/办公工具的清爽感） |
| run 态布局 | **方案 C**：顶部工具条常驻——表单压成一行（源文件夹 / 用户名 / TN / 开始重命名），始终可见可改 |
| 源文件夹选择交互 | 源文件夹格子**右侧内嵌一个小「选择」按钮**触发系统目录选择对话框 |
| 进度条 | **仅 running 时显示**（「处理中 N / 总」），完成后消失 |
| 「返回说明」按钮 | **移除**——工具条常驻，随时改参数重跑，不再需要手动切回说明 |
| 失败原因展示 | 保留在 ✗ 徽标的 `title`（hover 查看）；失败行淡红底 |
| 改动范围 | **纯前端** `index.html` / `style.css` / `main.js`；后端逻辑与数据契约不变 |

## 3. 视觉系统（浅色）

CSS 变量（替换现有深色变量；变量名尽量沿用以减少改动面）：

| 变量 | 值 | 用途 |
|------|----|----|
| `--bg` | `#f4f5f7` | 页面背景（浅灰） |
| `--surface` | `#ffffff` | 卡片/表面 |
| `--surface-2` | `#fafbfc` | 表头/输入框底/代码块 |
| `--fg` | `#1b1f27` | 主文字 |
| `--muted` | `#6b7280` | 次要文字 |
| `--faint` | `#9ca3af` | 占位符/弱化文字 |
| `--border` | `#e5e7eb` | 边框 |
| `--hairline` | `#f1f2f4` | 行分隔线 |
| `--accent` | `#2f6bff` | 主色（按钮、链接、强调） |
| `--accent-soft` | `#eef3ff` | 主色浅底（次级按钮、图标底） |
| `--success` | `#16a34a` | 成功 ✓ |
| `--error` | `#e5484d` | 失败 ✗ / 失败行 |
| `--error-soft` | `#fdeced` | 失败行底色 / 全局错误条底 |
| `--warn` | `#d97706` | 未识别金额 |

- 字体：`-apple-system, BlinkMacSystemFont, "SF Pro SC", "PingFang SC", sans-serif`，`-webkit-font-smoothing: antialiased`
- 圆角：卡片 12px、控件/按钮 8px
- 卡片阴影：`0 1px 2px rgba(16,24,40,.04)`（极轻）
- App 标题从原大号缩为 **16px**，副标题 12px `--faint`，让位给内容

## 4. 布局与三状态

整体结构（由 `.panel` 的 `data-mode` 驱动显隐，工具条贯穿所有状态常驻）：

```
┌ 顶部标题栏：ESI 发票重命名工具  ·  按 发票号 · 用户名 · TN 自动重命名
├ 工具条（常驻）：[📁 源文件夹…… (选择)] [用户名] [TN] [开始重命名]
├ (全局错误条：仅出错时，红色，工具条下方)
└ 主体区：
    · idle  → 使用说明卡（3 步 + 输出路径示例 + 底部小字）
    · run   → 结果卡（头部 + 汇总 banner + 进度条 + 表格）
```

**状态切换规则**：

| 状态 | 工具条 | 主体 | 进度条 | 打开文件夹键 |
|------|--------|------|--------|--------------|
| idle（首屏 / 点「返回」重置后） | 显示（占位态） | 使用说明 | 隐藏 | 隐藏 |
| running（执行中） | 显示（输入禁用、开始键转圈/禁用） | 结果卡，表格逐行填入、金额实时累加 | **显示**「处理中 N / 总」 | 隐藏 |
| 完成 | 显示（恢复可用） | 结果卡定格 | **消失** | 有成功输出时显示 |

## 5. 组件细节

### 5.1 工具条 `.toolbar`
- 一行 flex：源文件夹（`flex:2.4`）/ 用户名（`flex:1`）/ TN（`flex:1.3`）/ 开始按钮（固定宽）
- **源文件夹格子**：只读展示选中路径（占位「点击选择源文件夹」），格子右侧内嵌一个小「选择」按钮（`--accent-soft` 底）触发 `open({directory:true})`
- 用户名、TN 为常规输入框
- running 时：输入框/选择键禁用，开始按钮显示进行态

### 5.2 汇总 banner `.summary`
- 左：`价税合计总和` 小标签 + 大金额 `¥983.01`（34px，`¥` 较小、`--muted`；数字 `tabular-nums`）
- 右：4 个统计列（总张数 / 成功 / 失败 / 未识别金额），数字 18px；失败用 `--error`、未识别用 `--warn`，其余 `--fg`
- 金额与统计的数据来自后端 summary，运行中由 `renderRow` 实时累加、完成时 `updateSummary` 校准

### 5.3 进度条
- running 时显示在汇总与表格之间：细条 + 「处理中 N / 总」
- 完成后整条移除（不再保留满进度条）

### 5.4 表格（沿用现有数据组织）
- 列：`# / 源文件名 / 发票号 / 合计 / 结果`；列宽与对齐沿用（idx 36 / 发票号 170 mono / 合计 94 右对齐 tabular-nums / 结果 48 居中）
- 表头 `sticky` 浅灰底；行 `--hairline` 分隔；行 hover `--surface-2`
- 失败行 `--error-soft` 底、文件名/发票号转 `--error`
- 源文件名 `text-overflow: ellipsis` + `title` 看全名；✗ 的 `title` 放失败原因 `note`
- 无金额：合计列显示「—」`--faint`

### 5.5 全局错误条
- 源目录不存在 / 输出目录创建失败等后端 `Err`：工具条下方一条 `--error-soft` 底红字提示；不写入表格

### 5.6 使用说明卡（idle）
- 标题「使用说明」+ 3 步（圆形 accent 序号 + 文字）+ 「输出路径」代码块（占位 `<TN>`/`<发票号>` 等用 `--accent` 高亮）+ 底部小字「复制不移动 · 同名自动加序号 · 仅扫描顶层 PDF」

## 6. 数据契约（零后端改动的依据）

新界面所需数据，现有后端结构已完全覆盖：

| 界面位置 | 数据来源 |
|----------|----------|
| 工具条三个输入 | 前端表单自有（localStorage 记住用户名沿用） |
| 汇总大金额 | `summary.totalAmountCents`（+ 运行中 `row.amountCents` 累加） |
| 汇总统计 | `summary.total / success / failed / amountMissing` |
| 表格行 | `row.index / total / sourceName / invoiceNumber / amountDisplay / status / note` |
| 进度条 | `row.index / row.total` |
| 打开文件夹 | `summary.outputDir` |

→ `commands.rs` / `renamer.rs` / `pdf_parser.rs` **不需要改动**。

## 7. 前端改动清单

### 7.1 `index.html`
- 标题栏缩小
- `form` 卡 → 常驻 `.toolbar`（源文件夹格子含「选择」键 + 用户名 + TN + 开始按钮）
- 结果区：保留 `#summaryBar`（重构为大金额 + 4 统计）、`#progressTrack`、表格；**移除** `#logBox` 残留与「返回说明」按钮 `#clearLogBtn`
- 移除 idle 态会残留显示的进度文字结构（progress 仅 running 显示）

### 7.2 `style.css`
- 深色变量 → 第 3 节浅色变量
- 新增 `.toolbar` / 汇总 banner / 统计 / 进度条样式；表格与失败行改浅色
- 使用说明卡浅色化；移除已废弃的深色 `.log-*` 样式

### 7.3 `main.js`
- `setMode`：idle / run 之外，明确「完成时隐藏进度条」
- 进度条：running 显示并更新「处理中 N/总」，完成（`updateSummary`）后隐藏
- `renderRow` / `updateSummary`：数据契约不变，仅调整生成的 DOM 结构与 class
- 源文件夹：格子内「选择」键绑定 `open({directory:true})`（沿用现有 `pickDir` 逻辑）
- 移除「返回说明」按钮及其逻辑；重跑时 `resetTable` 清空表格与累加器
- 全局错误：`showError` 渲染到工具条下方错误条
